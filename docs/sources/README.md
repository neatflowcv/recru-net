# 개발자 채용공고 Source 정리

이 문서는 개발자 채용공고를 수집/탐색할 때 참고할 수 있는 주요 채용 플랫폼 정보를 정리한 자료입니다.

## 우선 추적 대상 (국내)

| Source | URL | 개발자 공고 특화도 | 특징 | 수집/연동 메모 |
|---|---|---|---|---|
| 점핏 (Jumpit) | <https://jumpit.saramin.co.kr/> | 매우 높음 | 개발 직군 중심, 기술스택/직무 태그가 잘 보임 | 개발자 공고 비중이 높아 1순위 소스로 적합 |
| 사람인 | <https://www.saramin.co.kr/> | 중간~높음 | 국내 대형 채용 플랫폼, 공고 수가 많음 | 공고 모수가 크므로 필터링(직군/기술스택) 전략 필요 |
| 원티드 (Wanted) | <https://www.wanted.co.kr/> | 높음 | IT/스타트업 공고가 많고 직무 분류가 비교적 명확 | 기업/포지션 메타데이터 구조 확인 후 수집 룰 설계 권장 |
| 로켓펀치 | <https://www.rocketpunch.com/> | 높음 | 스타트업 중심 공고/기업 정보 강점 | 스타트업 채용 탐색에 유리 |
| 프로그래머스 채용 | <https://career.programmers.co.kr/> | 매우 높음 | 개발자 채용 전용 성격, 직무/스택 기반 탐색 편리 | 개발 직군 신호가 명확해 정밀 매칭에 유리 |
| 잡코리아 | <https://www.jobkorea.co.kr/> | 중간 | 대형 채용 플랫폼, 전체 산업군 커버 | 개발자 전용은 아니므로 검색식/카테고리 튜닝 필요 |

| 벤처/스타트업 채용 페이지 모음 | (개별 기업 커리어 페이지) | 특정 기업 타깃 수집 |

## 권장 운영 방식

1. 1차 수집 소스는 `점핏 + 프로그래머스 + 원티드` 중심으로 구성
2. 대량 모수 확보는 `사람인 + 잡코리아`로 보완
3. 스타트업 특화 보완 소스로 `로켓펀치` 추가
4. 중복 제거 키는 `회사명 + 공고제목 + 근무지 + 게시일` 조합 우선 적용
5. 기술스택/직무명 정규화 사전을 별도로 유지 (`React`, `Spring`, `Node.js` 등)

## 체크리스트

- 각 사이트 이용약관/robots.txt를 확인하고 허용 범위 내에서 수집
- 요청 빈도 제한(레이트 리밋)과 재시도 정책을 적용
- 공고 만료/마감 상태를 주기적으로 동기화
- 개인정보(담당자 연락처 등) 저장 여부를 정책으로 명확히 제한
